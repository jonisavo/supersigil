## Overview

Implementation tasks for the `supersigil-verify` crate, ordered by dependency. The crate is built bottom-up: types and severity first, then individual rules, then the pipeline, hooks, report formatting, and finally the `affected` query.

## Phase 1: Foundation

</Task>
