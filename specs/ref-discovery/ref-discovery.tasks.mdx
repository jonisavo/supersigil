## Overview

Implementation follows TDD and the layer ordering from the design: graph
primitives first (foundation), then CLI command (surface), then verify
suggestion (integration). Each task writes tests before implementation.

<Task id="task-1" status="done" implements="ref-discovery/req#req-1-1, ref-discovery/req#req-1-2">
  Add `criteria()` and `criteria_by_fragment()` to `DocumentGraph`.

  TDD sequence:
  1. Write unit tests in `supersigil-core` asserting `criteria()` yields all
     referenceable components and `criteria_by_fragment()` finds matches across
     documents and returns empty for unknown fragments.
  2. Implement both methods as thin scans over the existing `component_index`.
  3. Verify with `cargo nextest run -p supersigil-core`.
</Task>

<Task id="task-2" status="done" depends="task-1" implements="ref-discovery/req#req-2-1, ref-discovery/req#req-2-3, ref-discovery/req#req-2-4, ref-discovery/req#req-2-5">
  Add the basic `refs` CLI command with `--format terminal|json` and `--all`.

  TDD sequence:
  1. Add `refs` to the clap command tree and write a clap parse test.
  2. Write integration tests in `cmd_refs.rs` for `--all` terminal output,
     JSON output shape, and empty-criteria case.
  3. Implement `refs.rs` command using `graph.criteria()`, sorted output, and
     the standard format dispatch.
  4. Verify with `cargo nextest run -p supersigil-cli`.
</Task>

<Task id="task-3" status="done" depends="task-2" implements="ref-discovery/req#req-2-2">
  Add prefix filtering to the `refs` command.

  TDD sequence:
  1. Write integration test: `supersigil refs auth/` shows only auth-prefixed
     refs.
  2. Implement prefix filtering in the command handler.
</Task>

<Task id="task-4" status="done" depends="task-2" implements="ref-discovery/req#req-3-1, ref-discovery/req#req-3-2, ref-discovery/req#req-3-3">
  Add Context_Scope resolution using TrackedFiles globs and cwd.

  TDD sequence:
  1. Write unit tests for `cwd_matches_glob` / `resolve_context_scope`:
     directory-prefix matching, non-matching cwd, empty TrackedFiles.
  2. Write integration tests: default scoping from cwd inside a tracked area,
     fallback to all when no TrackedFiles match, `--all` bypasses scoping.
  3. Implement scope resolution reusing `glob::Pattern` and
     `graph.all_tracked_files()`.
  4. Add stderr hint when context scoping is active.
</Task>

<Task id="task-5" status="done" depends="task-1" implements="ref-discovery/req#req-4-1, ref-discovery/req#req-4-2">
  Enhance the coverage rule to suggest full refs when unresolved evidence exists.

  TDD sequence:
  1. Write unit tests in `rules/coverage.rs`: unresolved evidence with matching
     fragment produces a "did you mean" suggestion; non-matching fragment keeps
     the original message; no unresolved evidence keeps the original message.
  2. Add `unresolved_evidence()` to `ArtifactGraph` (returns records with empty
     targets).
  3. Update the coverage rule to call `criteria_by_fragment` on the graph when
     unresolved evidence is present.
  4. Verify with `cargo nextest run -p supersigil-verify`.
</Task>
