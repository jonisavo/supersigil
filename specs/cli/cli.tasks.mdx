## Overview

Implementation tasks for the full CLI spec. Organized into phases by dependency order. Phase 1 covers cross-cutting infrastructure. Phases 2-4 add new commands. Phase 5 retrofits existing commands to the new output standards.

Existing implemented commands: `lint`, `ls`, `schema`, `context`, `plan`, `import`.
New commands to implement: `verify`, `status`, `affected`, `graph`, `init`, `new`.

  Each token maps to a semantic color (DocIdâ†’cyan, Statusâ†’yellow, etc.).
  `ColorConfig::paint()` returns `Painted` with ANSI escapes when color

## Phase 7: Bugfixes

<Task id="task-7-1" status="pending" depends="task-3-1" implements="cli/req#req-9-1">
  Fix `status` project-wide criteria counting. The current code iterates only
  top-level `doc.components`, but `Criterion` components are nested inside
  `AcceptanceCriteria` wrappers. Add a recursive helper to walk children, matching
  how `supersigil-core`'s `extract_criteria` works. This causes `criteria_total: 0`
  for all standard requirement documents.
</Task>

<Task id="task-7-2" status="pending" depends="task-4-2" implements="cli/req#req-13-2">
  Fix `new` command templates to use valid MDX syntax. Replace HTML comments
  (`&lt;!-- ... --&gt;`) with MDX comments (`{/* ... */}`) in the requirement and
  tasks scaffolds. The MDX parser rejects HTML comments inside JSX component
  bodies, so generated files fail `supersigil lint` immediately.
</Task>

<Task id="task-7-3" status="pending" depends="task-4-2" implements="cli/req#req-13-2">
  Fix `new` command templates to not emit empty `refs=""` placeholders. The design
  and property scaffolds currently emit `&lt;Validates refs="" /&gt;` which causes
  `split_list_attribute("")` to return a `BrokenRef` graph error, breaking all
  graph-based commands. Remove the refs attribute entirely or omit the component
  until the user fills it in.
</Task>

<Task id="task-7-4" status="pending" depends="task-7-3" implements="cli/req#req-13-2">
  Fix `new` command property template to include the required `strategy` attribute
  on `VerifiedBy`. The current template emits `&lt;VerifiedBy tag="" /&gt;` which
  fails lint with "missing required attribute 'strategy'". Use
  `&lt;VerifiedBy strategy="test" tag="" /&gt;` or omit the component.
</Task>

<Task id="task-7-5" status="pending" depends="task-2-2" implements="cli/req#req-14-4">
  Fix `supersigil-verify`'s `format_terminal` to respect Unicode/ASCII mode. The
  formatter currently hardcodes Unicode symbols (`âś” âś– âš  â„ą`) regardless of the
  `use_color` flag. Add a `use_unicode` parameter (or accept the full display
  config) and fall back to ASCII when unicode is disabled, matching the rest of
  the CLI's output conventions.
</Task>
