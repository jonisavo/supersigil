Supersigil provides editor extensions that connect to the `supersigil-lsp` language server, giving you live diagnostics, go-to-definition, autocomplete, and hover documentation directly in your editor.

## What you get

- **Live diagnostics** — parse errors, broken refs, missing coverage, and structural issues appear as you type and save.
- **Go-to-definition** — jump from a ref like `auth/req#req-1-1` to the criterion's source location.
- **Autocomplete** — document IDs, criterion IDs, component names with attribute snippets, and context-sensitive `status`/`strategy` values.
- **Hover documentation** — hover over a ref to see the target's title, body text, and a clickable link to navigate there.
- **Spec Explorer** — a sidebar tree view showing all spec documents grouped by feature area, with status indicators and click-to-open navigation.

## Installation

All editors require the `supersigil-lsp` binary:

```bash
cargo install supersigil-lsp
```

Then install the extension for your editor:

<Tabs>
<TabItem label="VS Code">

Build and install the extension from the repository:

```bash
cd editors/vscode
pnpm install
pnpm run package
code --install-extension supersigil-0.1.0.vsix
```

<Aside>
  The extension will be published to the VS Code Marketplace in a future release. For now, install from the built `.vsix` file.
</Aside>

Open any folder containing a `supersigil.toml`. The extension activates automatically and starts the language server. You should see **Supersigil** in the status bar.

</TabItem>
<TabItem label="IntelliJ IDEA">

Build and install the plugin from the repository:

```bash
cd editors/intellij
./gradlew buildPlugin
```

Then in IntelliJ: **Settings > Plugins > gear icon > Install Plugin from Disk...** and select `build/distributions/supersigil-intellij-0.1.0.zip`. Restart when prompted.

<Aside>
  The plugin will be published to the JetBrains Marketplace in a future release. For now, install from the built `.zip` file.
</Aside>

Open any project containing a `supersigil.toml` and open a `.md` or `.mdx` file. The plugin starts the language server automatically. The **Supersigil** tool window appears in the right sidebar.

The plugin requires IntelliJ IDEA 2025.3 or later (the unified distribution). It works with all IntelliJ-based IDEs that include the LSP client module.

</TabItem>
</Tabs>

## Configuration

Both extensions have one setting for the LSP server binary path:

| Editor | Setting | Default | Description |
|--------|---------|---------|-------------|
| VS Code | `supersigil.lsp.serverPath` | `null` | Absolute path to `supersigil-lsp`. |
| IntelliJ | Settings > Tools > Supersigil | empty | Absolute path to `supersigil-lsp`. |

When not set, the extension searches `$PATH` and common install locations (`~/.cargo/bin`, `~/.local/bin`). If the server binary is not found, a notification appears with install instructions.

Both extensions provide the full LSP feature set. A few features differ by editor:

| Feature | VS Code | IntelliJ |
|---------|---------|----------|
| Diagnostics | On type + save | On save |
| Completions | Yes | Yes |
| Hover | Yes | Yes |
| Go-to-definition | Yes | Yes |

## VS Code-specific features

### Multi-root workspaces

The VS Code extension supports multi-root workspaces. Each workspace folder with a `supersigil.toml` gets its own LSP server instance. The status bar shows the instance count (e.g. "Supersigil (2)") when multiple roots are active. Click it to see per-root status and actions.

### Commands

| Command | Description |
|---------|-------------|
| **Supersigil: Verify** | Run the verification pipeline and publish diagnostics. |
| **Supersigil: Restart Server** | Stop and restart all LSP server instances. |
| **Supersigil: Show Status** | Open a status menu showing per-root server state and actions. |

## IntelliJ-specific features

### Spec Explorer tool window

The Supersigil tool window in the right sidebar shows all spec documents grouped by project and feature prefix. Click a document to open it. The toolbar includes a verify button.

### Verify action

Available from **Tools > Supersigil: Verify** or the Spec Explorer toolbar button.

## Diagnostics tier

The LSP server runs at the `verify` diagnostic tier by default, which includes coverage checks. You can change this in `supersigil.toml`:

```toml
[lsp]
diagnostics = "lint"  # "lint" or "verify"
```

- **`lint`** — fast: parse errors and structural rules only.
- **`verify`** — full: adds evidence discovery and coverage analysis on save.

## Other editors

The `supersigil-lsp` binary communicates over stdio and implements the Language Server Protocol. It works with any LSP-capable editor:

- **Neovim** — configure via `nvim-lspconfig` with `cmd = {"supersigil-lsp"}` and `filetypes = {"markdown", "mdx"}`.
- **Zed** — add an LSP entry in settings pointing to the binary.
- **Helix** — add to `languages.toml` under the `markdown` language.

Editor-specific extensions for these editors are not yet available but the raw LSP works today.
